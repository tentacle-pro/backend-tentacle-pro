# 开发 - api.tentacle.pro 内容发布API服务需求文档 2026Q1

## 1. 背景与目标

### 1.1 背景
为了实现高效的公众号写作和发布能力，Obsidian 作为写作工作台，借助编程智能体调用不同的Agent Skills形成客户端双轨能力：
- 社区版Agent Skills：本地直连微信公众号API实现上传封面图和章节插图，并把图文文章直接保存到公众号后台草稿箱。
- 企业版Agent Skills：通过 api.tentacle.pro 统一提供 Markdown 转 HTML 与草稿发布接口。
	- 1) 实现为每家客户提供个性的排版渲染模板。
	- 2) 实现社区版功能的同时通过转发数据避免频繁更新公众号后台IP白名单(始终使用本服务器IP)。
	- 3) 企业客户的AppID/AppSecret配置在服务器端，以应对“系统自动创作内容并发布到草稿箱”的自动化内容运营需求。

### 1.2 目标
在 api.tentacle.pro 上实现稳定、可观测、可扩展的企业 API 服务，满足：
1) Markdown 转微信兼容 HTML 子集
2) 微信草稿箱发布（封面图 + 正文插图）
3) API_KEY 到公众号凭据映射
4) 服务端 token 缓存与自动刷新
5) 全流程草稿箱模式（不做群发）

### 1.3 非目标
- 不实现群发接口
- 不实现多租户计费系统（仅预留字段）
- 不实现复杂 CMS 系统（仅实现轻量管理员后台：客户接入、模板与素材管理）

## 2. 范围与需求

### 2.1 功能需求
- F1: 提供 markdown2html 接口
- F2: 提供封面图上传（微信永久素材）接口
- F3: 提供正文图片上传（微信 uploadimg）接口
- F4: 提供草稿新增接口
- F5: 服务端维护并刷新公众号 access_token
- F6: API_KEY 鉴权与账号映射
- F7: 请求日志、错误日志、审计日志

### 2.2 非功能需求
- N1: 可用性目标 >= 99.9%
- N2: API p95 延迟 < 800ms（不含微信上游抖动）
- N3: 统一错误码与可追踪 request_id
- N4: 所有密钥与 token 加密存储
- N5: 支持水平扩展（无状态应用层）

## 3. 技术栈与工程约束

## 3.1 指定技术栈
- 运行时：Bun
- 语言：TypeScript
- Web 框架：Hono
- ORM：Drizzle ORM
- 数据库：PostgreSQL

### 3.2 推荐工程结构
- apps/api：Hono API 服务
- packages/core：领域模型、DTO、错误码、工具函数
- packages/db：Drizzle schema、migrations、seeds
- packages/wechat：微信 API 客户端与 token 管理

### 3.3 编码规范
- TypeScript strict 模式
- 输入参数统一使用 schema 校验（建议 zod）
- 错误处理统一为结构化响应
- 任何对微信上游调用都必须可重试、可观测

## 4. 系统架构设计

### 4.1 总体架构
1) Client（Obsidian project level Agent Skills）携带 API_KEY 调用 api.tentacle.pro
2) API Gateway/应用层通过 API_KEY 映射 tenant/account
3) Token Service 获取或刷新微信 access_token
4) WeChat Adapter 调用微信素材/草稿接口
5) PostgreSQL 存储租户、账号、token、审计日志

### 4.2 核心模块
- Auth 模块：API_KEY 校验、租户识别、权限判定
- Render 模块：Markdown -> HTML 渲染服务编排
- WeChat Publish 模块：图片上传与草稿发布
- Token 模块：缓存、续期、失效重试
- Audit 模块：请求追踪与审计留痕

## 5. API 规范（V1）

### 5.1 通用约定
- Base URL: https://api.tentacle.pro
- Header:
  - Authorization: Bearer <API_KEY>
  - X-API-Key: <API_KEY>
  - Content-Type: application/json 或 multipart/form-data
- 响应统一字段：
  - ok: boolean
  - request_id: string
  - error?: { code, message, details? }
  - data?: any

### 5.2 POST /markdown2html
用途：将 Markdown 转为微信兼容 HTML。

说明：模板为平台全局共享资源，不与企业做绑定；任意有效 API_KEY 均可通过 templateId 使用任意模板。

模板机制（2026-03 重构版）：
- 平台预置 3 套模板：经典（classic）、优雅（grace）、简洁（simple）。
- 预置模板以 Tailwind 规则存储在数据库 `templates.config` 中，并标记为 `config.meta.locked = true`。
- 预置模板不可直接修改，管理员需“另存为模板”后在 Web 端持续调整 Tailwind 规则。
- 复制后的模板自动标记 `config.meta.locked = false`，可直接保存、改名、删除。

请求体：
- markdown: string（必填）
- templateId: string（必填）
- title?: string

成功响应：
- ok: true
- data.html: string

### 5.3 POST /post2wechat/upload/permanent-image
用途：上传封面图（永久素材），返回 media_id。

请求：multipart/form-data
- media: file（二选一）
- image_url: string（二选一）

成功响应：
- ok: true
- data.media_id: string

### 5.4 POST /post2wechat/upload/article-image
用途：上传正文图，返回微信可访问 URL。

请求：multipart/form-data
- media: file（二选一）
- image_url: string（二选一）

成功响应：
- ok: true
- data.url: string

### 5.5 POST /post2wechat/draft/add
用途：新增草稿。

请求体：
- title: string（必填）
- author?: string
- digest?: string
- content: string（必填）
- thumb_media_id: string（必填）
- need_open_comment?: 0|1（默认 1）
- only_fans_can_comment?: 0|1（默认 0）

成功响应：
- ok: true
- data.media_id: string

## 7. 安全与合规

- API_KEY 仅保存哈希（不可逆）
- app_secret、access_token 必须加密存储（KMS 或应用层 AES-GCM）
- 全站 HTTPS，拒绝明文回源
- 接口限流（按 client_id + endpoint）
- 日志脱敏（禁止输出 app_secret、token、原始 API_KEY）

## 8. token 管理策略

- 命中有效 token：直接使用
- token 即将过期（如 <10 分钟）：后台异步刷新
- 微信返回 token 失效：触发一次强制刷新并重试请求
- 刷新并发控制：基于 wechat_account_id + version 锁

## 9. 错误码规范

- AUTH_401_INVALID_API_KEY
- AUTH_403_BINDING_NOT_FOUND
- WX_400_UPLOAD_FAILED
- WX_401_TOKEN_INVALID
- WX_429_RATE_LIMIT
- RENDER_400_INVALID_TEMPLATE
- SYS_500_INTERNAL_ERROR

## 10. 可观测性与运维

- 结构化日志：JSON 输出，包含 request_id/client_id/endpoint
- 指标：QPS、错误率、p95/p99、微信上游失败率
- Trace：建议 OpenTelemetry
- 健康检查：/healthz（进程）、/readyz（数据库与关键依赖）
